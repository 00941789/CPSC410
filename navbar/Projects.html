My tester and a simple test file are given above. In general these are the problems I saw:

  • Changing the case of utilities.h. Utilities.h and utilities.h are different files
  • Choosing descriptive variable names. For instance, between 'bro' and 'processStats'. Which is a better description of a container holding process_stats structs?
  • Includeing user includes before system includes (should be the other way around)
  • Leaving print statements in code. Makes the solution look a bit rushed and rubbishy. If you insists on print statements, use a logging library instead.
  • Leaving an unseemly amount of whitespace and/or commentted out code in your solution. Makes an instant bad impression.

3 3/10/19 NOTE: This (and all following) project requires C++ 11 support. It is already enabled for this application (assuming you are using the GNU toolchain). In general, to enable C++11 support, click on the project name->Properties-> C/C++ Build->Settings->GCC C++ Compiler-> Dialect ->'Language Standard' dropdown. Then choose at least ISO C++11(-std=c++0x). Make sure you include the pthread library under GCC C++ linker->Libraries. Add pthread in the Libraries (-l) pane on the right.
